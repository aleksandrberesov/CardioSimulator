# Протокол TCP CardioSimulator (JSON)

Этот документ описывает структуру JSON-протокола, используемого для обмена командами управления ЭКГ и данными сигналов через TCP-соединение.

## Общие характеристики

- **Кодировка**: UTF-8.
- **Обрамление (Framing)**: Одно сообщение на строку, завершающееся символом `\n` (LF). Пустые строки игнорируются.
- **Тип данных**: JSON-объекты.

## Базовая структура (Envelope)

Каждое сообщение является JSON-объектом и содержит следующие общие поля:

| Поле   | Тип    | Обязательно | Описание |
| :---   | :---   | :---        | :---     |
| `type` | string | Да          | Тип сообщения. Определяет структуру остальных полей. |
| `id`   | string | Нет         | Произвольный идентификатор сообщения. Используется для сопоставления ответов. |

---

## Реализованные типы сообщений

### 1. `start` — Запуск потока данных
Отправляется для начала генерации/передачи сигнала.

| Поле         | Тип    | Обязательно | Описание |
| :---         | :---   | :---        | :---     |
| `sampleRate` | int    | Нет         | Частота дискретизации (Гц). |
| `params`     | object | Нет         | Словарь дополнительных параметров (строковые ключ-значение). |

**Пример:**
```json
{"type":"start", "id":"m1", "sampleRate":250, "params":{"mode":"diagnostic"}}
```

### 2. `stop` — Остановка потока
Останавливает передачу данных. Дополнительных полей нет.

**Пример:**
```json
{"type":"stop", "id":"m2"}
```

### 3. `points` — Передача отсчетов сигнала
Содержит массив значений амплитуд для конкретного отведения.

| Поле     | Тип            | Обязательно | Описание |
| :---     | :---           | :---        | :---     |
| `lead`   | string         | Нет         | Токен отведения (напр., "I", "II", "V1"). |
| `identy` | string         | Нет         | Идентификатор источника/серии данных. |
| `offset` | int            | Нет         | Индекс (смещение) первого отсчета в серии (дефолт: 0). |
| `values` | array<float\> | Да          | Список числовых значений амплитуд. |

**Пример:**
```json
{"type":"points", "lead":"II", "offset":500, "values":[0.12, 0.15, 0.21]}
```

### 4. `upload` — Загрузка файла
Инициализирует передачу бинарного файла (например, `Pathologies.zip`).
**Важно**: Сразу за символом `\n` после этого JSON-объекта должны следовать сырые бинарные данные объемом `size` байт.

| Поле       | Тип    | Обязательно | Описание |
| :---       | :---   | :---        | :---     |
| `filename` | string | Да          | Имя передаваемого файла. |
| `size`     | long   | Да          | Точный размер бинарного контента в байтах. |

**Пример:**
```json
{"type":"upload", "filename":"data.zip", "size":1048576}
```

### 5. `ack` — Подтверждение приема
Ответ сервера на успешную загрузку файла.

| Поле       | Тип    | Обязательно | Описание |
| :---       | :---   | :---        | :---     |
| `filename` | string | Да          | Имя сохраненного файла. |
| `bytes`    | long   | Да          | Количество фактически записанных байт. |

**Пример:**
```json
{"type":"ack", "filename":"data.zip", "bytes":1048576}
```

---

## Список токенов отведений (Leads)
В поле `lead` используются следующие значения (регистронезависимо):
`I`, `II`, `III`, `aVR`, `aVL`, `aVF`, `V1`, `V2`, `V3`, `V4`, `V5`, `V6`.
