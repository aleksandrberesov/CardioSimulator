# Текущая структура файла .dat

Этот документ содержит краткий обзор структуры файлов `.dat`, используемых в приложении CardioSimulator. Данный формат разработан для обеспечения эффективности и простоты, объединяя все 12 стандартных отведений ЭКГ для одной патологии в один текстовый файл в кодировке UTF-8.

## 1. Спецификация файла

*   **Кодировка**: UTF-8
*   **Окончания строк**: LF (`\n`)
*   **Формат**: Обычный текст с парами `ключ:значение`.
*   **Блоки**: Разделяются одной или более пустыми строками.

## 2. Описание структуры

Типичный файл патологии `.dat` состоит из одного **Блока заголовка** (Header Block), за которым следуют несколько **Блоков отведений** (Lead Blocks).

### 2.1 Блок заголовка
Содержит метаданные о патологии.

| Ключ | Описание | Пример |
| :--- | :--- | :--- |
| `pathology` | Уникальный идентификатор (slug) | `pathology:tachpm` |
| `title` | Название на английском языке | `title:Atrial tachycardia` |
| `name` | Название на русском языке (UTF-8) | `name:Предсердная тахикардия` |
| `leads` | Общее количество блоков отведений в файле | `leads:12` |
| `markers` | (Опционально) Индексы контрольных точек ЭКГ | `markers:120:P_START,225:R_PEAK` |

### 2.2 Блок отведения
Один блок на каждое отведение (обычно всего 12 блоков).

| Ключ | Описание | Пример |
| :--- | :--- | :--- |
| `lead` | Идентификатор отведения (I, II, III, aVR, aVL, aVF, V1-V6) | `lead:II` |
| `count` | Количество целых чисел в поле `points` | `count:2782` |
| `points` | Разделенные запятыми необработанные значения АЦП | `points:1024,1024,1025,...` |

## 3. Семантика данных

*   **Изолиния (Baseline)**: Все значения `points` центрированы относительно изоэлектрической линии со значением **1024**.
*   **Значения отсчетов**: 32-битные целые числа со знаком (обычно в диапазоне `int16`).
*   **Отсутствующие отведения**: Если отведение отсутствует (например, в патологии `emd`), соответствующий блок полностью опускается.
*   **Канонический порядок**: `I, II, III, aVR, aVL, aVF, V1, V2, V3, V4, V5, V6`.

## 4. Маркеры контрольных точек
Маркеры в заголовке используют формат `индекс:ТИП`.
*   **Индекс**: Индекс отсчета (начиная с 0).
*   **Типы**:

| Тип | Описание |
| :--- | :--- |
| `P_START` | Начало зубца P |
| `P_PEAK` | Пик зубца P |
| `P_END` | Конец зубца P |
| `QRS_START` | Начало комплекса QRS |
| `Q_PEAK` | Пик зубца Q |
| `R_PEAK` | Пик зубца R |
| `S_PEAK` | Пик зубца S |
| `QRS_END` | Конец комплекса QRS |
| `T_START` | Начало зубца T |
| `T_PEAK` | Пик зубца T |
| `T_END` | Конец зубца T |

## 5. Пример использования

```text
pathology:normal
title:Normal Sinus Rhythm
name:Нормальный синусовый ритм
leads:12
markers:100:P_START,125:P_PEAK,140:P_END,180:QRS_START,195:R_PEAK,210:QRS_END

lead:I
count:1000
points:1024,1024,1024,1024,1025,1027,...

lead:II
count:1000
points:1024,1024,1024,1026,1029,1035,...

...
```
